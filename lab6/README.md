# Lab 6: ศึกษาและฝึกปฏิบัติ เกี่ยวกับ Navigation and routing ของ Flutter

โปรเจกต์ Flutter นี้เป็นส่วนหนึ่งของการทดลอง โดยมีวัตถุประสงค์เพื่อสาธิตการใช้งานวิดเจ็ตต่างๆ ของ Material Design ใน Flutter แอปพลิเคชันนี้ประกอบด้วยหน้าต่างๆ ที่จัดหมวดหมู่ตามประเภทของวิดเจ็ต เพื่อให้เห็นถึงวิธีการใช้งานและการทำงานของวิดเจ็ตเหล่านั้น

## โครงสร้างและการทำงานของแอปพลิเคชัน

แอปพลิเคชันเริ่มต้นที่ `HomePage` ซึ่งทำหน้าที่เป็นเมนูหลัก แสดงรายการหมวดหมู่ของวิดเจ็ตต่างๆ ผู้ใช้สามารถเลือกหมวดหมู่ที่สนใจเพื่อนำทางไปยังหน้ารายละเอียดของหมวดหมู่นั้นๆ

### หน้าต่างๆ ในแอปพลิเคชัน:

1.  **CommunicationPage**:
    *   **วัตถุประสงค์**: สาธิตการใช้วิดเจ็ตเพื่อการสื่อสารกับผู้ใช้
    *   **วิดเจ็ตที่ใช้**:
        *   `SnackBar`: แสดงข้อความสั้นๆ ที่ด้านล่างของหน้าจอเมื่อผู้ใช้กดปุ่ม "Show Snackbar"
        *   `AlertDialog`: แสดงกล่องโต้ตอบ (Dialog) ที่มีข้อความและปุ่ม "OK" เพื่อปิดกล่องโต้ตอบนั้น

2.  **ContainmentPage**:
    *   **วัตถุประสงค์**: สาธิตการใช้วิดเจ็ตสำหรับจัดกลุ่มหรือครอบเนื้อหา
    *   **วิดเจ็ตที่ใช้**:
        *   `Card`: แสดงการ์ดที่มีเงา (elevation) ใช้สำหรับจัดกลุ่มข้อมูลที่เกี่ยวข้องกันให้ดูเป็นสัดส่วนและสวยงาม ภายใน `Card` มี `Container` และ `Text`

3.  **NavigationPageDemo**:
    *   **วัตถุประสงค์**: สาธิตการทำงานของระบบนำทาง (Navigation) เบื้องต้น
    *   **การทำงาน**:
        *   มีปุ่ม "Go to Second Page" ซึ่งเมื่อกดแล้วจะใช้ `Navigator.push()` เพื่อเปิดหน้าใหม่ (`SecondPage`) ขึ้นมาซ้อนทับหน้าปัจจุบัน
        *   ใน `SecondPage` จะมีปุ่ม "Back" ซึ่งใช้ `Navigator.pop()` เพื่อย้อนกลับไปยังหน้าก่อนหน้า

4.  **SelectionPage**:
    *   **วัตถุประสงค์**: สาธิตการใช้วิดเจ็ตที่ให้ผู้ใช้เลือกตัวเลือกต่างๆ
    *   **วิดเจ็ตที่ใช้**:
        *   `CheckboxListTile`: สำหรับการเลือก/ไม่เลือก (true/false) ในรูปแบบรายการ
        *   `SwitchListTile`: สำหรับการเปิด/ปิด (on/off) ในรูปแบบรายการ
        *   `RadioListTile`: สำหรับการเลือกตัวเลือกเดียวจากกลุ่มของตัวเลือก (ในตัวอย่างคือการเลือกเพศ)

5.  **TextInputPage**:
    *   **วัตถุประสงค์**: สาธิตการรับข้อมูลประเภทข้อความจากผู้ใช้
    *   **วิดเจ็ตที่ใช้**:
        *   `TextField`: เป็นช่องสำหรับกรอกข้อความ ผู้ใช้สามารถป้อนชื่อลงไปได้
        *   `ElevatedButton` (Submit): เมื่อกดปุ่มนี้ ข้อความที่ผู้ใช้กรอกจะถูกนำมาแสดงผลที่วิดเจ็ต `Text` ด้านล่าง

## การนำทางและการจัดการเส้นทาง (Navigation and Routing)

โปรเจกต์นี้ใช้ระบบการนำทางที่เรียกว่า "Named Routes" ซึ่งเป็นวิธีที่แนะนำสำหรับการจัดการการนำทางในแอปพลิเคชันที่มีหลายหน้า

*   **การตั้งค่า**: ในไฟล์ `lib/main.dart` ภายใน `MaterialApp` จะมีการกำหนด `initialRoute` และ `routes`
    *   `initialRoute: '/'`: กำหนดให้หน้าแรกของแอปคือ `HomePage`
    *   `routes`: เป็น Map ที่จับคู่ระหว่างชื่อเส้นทาง (เช่น `'/communication'`) กับ WidgetBuilder ที่จะสร้างหน้า UI นั้นๆ (เช่น `CommunicationPage()`)

*   **การใช้งาน**: ใน `HomePage` เมื่อผู้ใช้แตะที่รายการเมนู `ListTile` จะมีการเรียกใช้ฟังก์ชัน `Navigator.pushNamed(context, route)` เพื่อนำทางไปยังหน้าที่กำหนดไว้ตามชื่อเส้นทาง (route) ที่ส่งเข้ามา

วิธีนี้ช่วยให้โค้ดการนำทางมีความสะอาดและจัดการได้ง่ายขึ้น โดยเฉพาะในโปรเจกต์ขนาดใหญ่ เพราะไม่ต้องสร้าง `MaterialPageRoute` ทุกครั้งที่ต้องการเปลี่ยนหน้า และสามารถจัดการเส้นทางทั้งหมดได้จากที่เดียว
